Входные данные:
{
  "task_struct": { ... },  // из PARSE, структура условия (ветка/тип/данные)
  "raw_task_text": "..." // из PARSE, условие задачи
  "raw_answer_text": "..." // из OCR, дословная рукопись (может быть пусто)
}

Требуемый выход ТОЛЬКО в формате JSON:
{
  "norm_task": {
    "kind": "<math|ru|generic>",
    "data": "<СТРОКА с JSON‑объектом payload выбранной семьи>"
  },
  "norm_answer": {
    "value": "<СТРОКА: простой ответ или СТРОКА с JSON {result,steps?,conflict?}>",
    "units": "<опционально>",
  }
}

Правила:
- НЕ пере‑парсивай условие. Используй только поля из task_struct и raw_task_text.
- Ответ извлекай из raw_answer_text без догадок. Нет уверенности — "value":"null".
- В "data" и "value" всегда передавай СТРОКУ. Если нужна структура — положи корректный JSON внутрь строки без переносов.
- Никаких лишних ключей и текста вокруг JSON.

Ограничения и минимальные payload:
- math → data: {condition?, question?, givens[], targets[], equations[], constraints[], checks[]}
- ru   → data: {condition?, question?, tokens[], targets[{index,expected,rule_ref?}], rule?, checks[]}
- generic → data: {summary}

Примеры (сокращённо):
// math
{
  "norm_task": {"kind":"math","data":"{\"givens\":[{\"symbol\":\"H\",\"value\":35},{\"symbol\":\"L\",\"value\":94}],\"equations\":[\"R+P=H\",\"4*R+2*P=L\"]}"},
  "norm_answer": {"value":"{\"result\":{\"rabbits\":12,\"pheasants\":23}}","units":"animals"}
}
// ru
{
  "norm_task": {"kind":"ru","data":"{\"tokens\":[\"скупой\",\"богач_\"],\"targets\":[{\"index\":1,\"expected\":\"ь\"}]}"},
  "norm_answer": {"value":"[\"богачь\"]"}
}
